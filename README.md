# Banking Transaction ETL Pipeline

## Overview

This project demonstrates an end-to-end ETL (Extract, Transform, Load) pipeline built using Python, SQL, pandas and GitHub.

The project focuses on improving the quality of banking transaction data by identifying and resolving common data quality issues before producing a clean dataset for reporting and analysis.

---

## Technologies Used

- Python
- Pandas
- SQL
- GitHub

---

## Dataset

The raw dataset contains banking transaction records with several data quality issues, including:

- Duplicate transaction IDs
- Missing customer IDs
- Negative transaction amounts
- Cancelled transactions

---

## ETL Process

### Extract
Loaded the raw banking transaction dataset.

### Transform
Applied business rules to:

- Remove duplicate transaction IDs
- Remove missing customer IDs
- Remove negative transaction amounts
- Remove cancelled transactions

### Load
Generated a clean output dataset named **clean_bank_transactions.csv**.

---

## Project Outcome

The final dataset contains only valid banking transactions and is suitable for reporting and further analysis.
